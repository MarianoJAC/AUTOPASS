# EXPLICACION DEL CODIGO - SISTEMA PARKINGTECH

ESTE DOCUMENTO DETALLA EL FUNCIONAMIENTO DEL SISTEMA DE GESTION DE ESTACIONAMIENTO INTELIGENTE.

## 1. ARQUITECTURA GENERAL
EL SISTEMA SE COMPONE DE TRES CAPAS PRINCIPALES:
*   **BACKEND (API):** CENTRALIZA LA LOGICA DE NEGOCIO Y LA BASE DE DATOS.
*   **SERVICIO ALPR:** SE ENCARGA DE LA CAPTURA DE VIDEO Y RECONOCIMIENTO DE TEXTO EN PATENTES.
*   **HARDWARE (ESP32):** CONTROLA LOS ELEMENTOS FISICOS (BARRERAS, SENSORES, PANTALLAS).

---

## 2. COMPONENTES TECNICOS

### BACKEND (MAIN.PY, MODELS.PY, DATABASE.PY)
*   **TECNOLOGIA:** FASTAPI Y SQLALCHEMY.
*   **LOGICA DE ACCESO:** CUANDO RECIBE UNA PATENTE DESDE EL SERVICIO ALPR, EL BACKEND VERIFICA:
    1.  SI EL VEHICULO YA ESTA DENTRO (EVITA DUPLICADOS).
    2.  SI EXISTE UNA RESERVA VIGENTE PARA ESA PATENTE Y ESE DIA/HORA.
    3.  SI NO HAY RESERVA, VERIFICA SI HAY LUGAR DISPONIBLE (MODO "AL PASO").
*   **COMUNICACION:** SI EL ACCESO ES VALIDO, ENVIA UN MENSAJE POR **MQTT** (UN PROTOCOLO LIVIANO DE MENSAJERIA) PARA QUE LA BARRERA SE ABRA.
*   **EGRESO:** AL DETECTAR UNA SALIDA, VERIFICA QUE LA ESTADIA ESTE PAGADA. SI NO LO ESTA, CALCULA EL COSTO SEGUN EL TIEMPO TRANSCURRIDO Y LA TARIFA CONFIGURADA.

### SERVICIO DE RECONOCIMIENTO (ALPR_SERVICE.PY)
*   **TECNOLOGIA:** OPENCV (PARA PROCESAMIENTO DE IMAGENES) Y EASYOCR (PARA LECTURA DE TEXTO).
*   **FUNCIONAMIENTO:**
    1.  CAPTURA EL VIDEO DE UNA CAMARA (O ARCHIVO).
    2.  PROCESA LOS CUADROS (FRAMES) PARA MEJORAR EL CONTRASTE.
    3.  BUSCA PATRONES QUE PAREZCAN PATENTES (FORMATOS MERCOSUR Y VIEJOS).
    4.  CUANDO LOGRA UN CONSENSO (VARIAS LECTURAS IGUALES), ENVIA LA PATENTE AL BACKEND.

### FIRMWARE DE HARDWARE (FIRMWARE_ESP32.INO)
*   **TECNOLOGIA:** C++ PARA ARDUINO/ESP32.
*   **FUNCIONAMIENTO:**
    1.  SE CONECTA A WIFI Y SE SUSCRIBE A UN TOPICO DE MQTT.
    2.  AL RECIBIR EL COMANDO "OPEN", ACTIVA UN **SERVOMOTOR** PARA LEVANTAR LA BARRERA Y MUESTRA UN MENSAJE EN UNA **PANTALLA LCD**.
    3.  UTILIZA UN **SENSOR ULTRASONICO** PARA DETECTAR CUANDO EL AUTO TERMINO DE PASAR Y ASI CERRAR LA BARRERA DE FORMA SEGURA.

---

## 3. FLUJO DE USUARIO TIPICO
1.  EL AUTO SE ACERCA A LA ENTRADA.
2.  LA CAMARA DETECTA LA PATENTE.
3.  EL BACKEND CONFIRMA LA VALIDEZ Y REGISTRA EL INGRESO.
4.  EL ESP32 ABRE LA BARRERA.
5.  EL AUTO ESTACIONA Y EL AFORO SE ACTUALIZA EN EL DASHBOARD.
6.  PARA SALIR, EL SISTEMA REPITE EL PROCESO PERO EXIGE QUE EL PAGO ESTE CONFIRMADO.

---

## 4. BASE DE DATOS
SE UTILIZA **SQLITE** CON LAS SIGUIENTES TABLAS:
*   **USERS/VEHICLES:** DATOS DE CLIENTES Y SUS AUTOS.
*   **RESERVATIONS:** CALENDARIO DE ACCESOS PROGRAMADOS.
*   **ACCESS_LOGS:** HISTORIAL DETALLADO DE ENTRADAS, SALIDAS Y FOTOS CAPTURADAS.
*   **PARKING_AFORO:** ESTADO ACTUAL DE LA OCUPACION.
